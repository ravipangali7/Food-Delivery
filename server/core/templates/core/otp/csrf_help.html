Why “CSRF cookie not set” or 403 on POST /api/…

Django’s CSRF protection expects a CSRF cookie (and matching token on unsafe requests) when the browser uses session authentication across origins. A JavaScript app on http://localhost:8080 calling http://127.0.0.1:8000 does not automatically get that cookie, so POST requests can fail with CSRF verification failed / CSRF cookie not set.

What we changed: the REST API defaults to token authentication only (no session auth on API views). Use Authorization: Token <key> after login, or call the API from the same site as Django so the browser can store the CSRF cookie.

This OTP page is served by Django on the same origin as the form POST and includes Django’s CSRF token in the form, so the browser sends a valid token — no extra setup.
